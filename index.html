Yo les négros, voila ce qu'il vous faut pour le projet

Installer Tomcat

C'est le serveur, alors ca peut être utile
Windows : http://apache.lehtivihrea.org/tomcat/tomcat-7/v7.0.52/bin/apache-tomcat-7.0.52-windows-x86.zip
Linux : sudo apt-get install tomcat

Tu extrait ca ou tu veux, j'men fous. La seule chose à connaitre, c'est ou se trouve le dossier webapps Sous Linux, par défaut, c'est dans /usr/share/tomcat7/webapps.

Installer Maven

Ca permet de compiler, alors c'est cool.
Windows : http://apache.lehtivihrea.org/maven/maven-3/3.2.1/binaries/apache-maven-3.2.1-bin.zip
Linux : sudo apt-get install maven

Tu extrais ca ou tu veux. La seule condition, c'est d'ajouter le dossier bin/ dans le PATH (pour windows).

Installer MySQL

Windows : http://dev.mysql.com/downloads/file.php?id=450945
Linux : sudo apt-get install mysql

Bon, c'est la que ca devient drôle. L'installeur va demander le mot de passe du compte root. Tu mets 'rootpassword', parce que sinon faut changer les scripts, et c'est chiant.

Une fois que c'est fait : Windows : tu vérifie qu'il y a bien MySQL Notifier dans la barre des taches, et qu'il est bien lancé Linux : sudo service mysqld start

Git puller

Si tu n'as JAMAIS configuré git, que tu n'as pas pull le dépot, etc.
Windows : http://git-scm.com/download/win (C'est vraiment, vraiment pratique. Une fois que Git Bash est lancé, c'est quasiment un environnement linux)
Linux : sudo apt-get install git

Dans le terminal, faut penser a préparer les clés SSH en tapant

  • eval `ssh-agent`
  • ssh-add ~/.ssh/exxxxxxxxxx

Si tu n'as pas encore pull le dépot (vilain) git clone git@gitinfo:appli_med_b.git

Sinon, tu te mets dans le dossier ou ca a déjà été cloné, puis git pull quoi.

Préparer MySQL

Une fois que c'est pull, tu te mets dans le dossier avkapp/mysql (avec ton terminal)
mysql -u root --password=rootpassword
Une fois que t'es log dans mysql :
source create_db.sql;
Une fois que c'est fait, tu quitte MySQL (exit, ou bien un bon vieux CtrlC)
On se relog, sur la base de données
mysql -u root --password=rootpassword avkapp

Préparation des données :
source create_tables.sql;
source insert_med.sql;
INSERT INTO Office(Name, Address, PhoneNumber) VALUES ("Test", "Test", "000000");
INSERT INTO Profile(Name) VALUES ("Medecin"), ("Test");

OK, on a maintenant une base de données utilisable

Lancer Tomcat

Windows : Dossier d'installation tomcat, bin/, tu ouvres ton petit terminal et tu tapes catalina.bat start
Linux : sude service tomcat7 start

Coder

Tu vas dans le dossier server que tu as pull. Si a l'interieur y'a un fichier pom.xml, c'est que t'es au bon endroit C'est dans ce dossier que se font TOUTES les modifications. Java, HTML, JS, CSS, tout.

Pour lancer la compilation, tu te mets dans le dosser serveur, et tu fais mvn clean package. Si t'as un beau message "BUILD SUCCESS", c'est que c'est bon. Sinon, bah tu sais pas coder, et tu regarde les erreurs. Une fois que c'est fait, ca génère un fichier avkapp.war dans target/ Tu prends ce fichier .war, tu le mets dans le dossier webapps. Tomcat est CENSE le déployer tout seul. Si tu vois que ca reste sur "Initializing Jersey Application...":

  • Tu fermes Tomcat
  • tu supprime le dossier webapps/avkapp
  • Tu relances Tomcat

Après, l'application est disponible à l'adresse localhost:8080/avkapp/
